    Personal Website

    My very own webpage

    In preparation for one of my courses at UBC, I picked up Jon Duckett's HTML and CSS: Design and Build Websites book from the nearest Amazon and created this page as a means to experiment with the front-end environment. As I got further into the book, I started forgetting what I had previously read in the book. So, I decided to transform the experimental webpage into my very own online profile page as a way to solidify what I have learned. What was originally supposed to be a throwaway page turned into this mini project!

    The Process

    I had quite the rough start. At the time, I've only ever coded in Java and C++, so HTML and CSS seemed extremely foreign to me. While I quickly familiarzed myself with HTML, CSS was an absolute pain. My lack of artistic sense coupled with my OCD did not help much, either. I still remember being extremely frustrated because I could not vertically center an element inside a div box even after hours of research on the internet. As I discovered more tools such as Flex Box and picked up JavaScript, I began recoding certain parts of the site and even went as far as to starting the page from scratch. My confidence in my front-end skills solidified further with every piece of code I rewrite. I'm still constantly making changes and am excited to try out some of the newer features provided by HTML6 and CSS4.

    What I learned

    It was an amazing experience. I picked up on a ton of things throughout this entire thing. I learned the basics of creating a responsive and interactive site. I learned about colour theory so that I won't dress like a Christmas tree in the future. I learned how to add interactivity using JavaScript/TypeScript. I relearned how to use Illustrator and Photoshop to create some SVG animations. Most importantly, I learned how to vertically center an element in a div box!

    Tools

    HTML, SCSS, TypeScript, Adobe Illustrator, Coffee

    Space Invaders and Brick Breaker

    An introduction to programming

    My first exposure to programming was back in 2012 when I was still in grade 10. My first assignment was to make a rectangle oscillate left and right using simple if statements. At the time, I was completely blown away because it completely changed my understanding of how programs and applications worked.

    Brick Breaker

    Brick Breaker was the very first game I attempted to recreate. I coded the game up using Visual Basic. By coding, I meant dragging and dropping a lot of shapes onto a form. I also had no idea what loops were at the time, so I had to copy and paste a lot of brick code while manually changing the (x,y) coordinates. I was so into programming my game that I pulled a few all-nighters implementing various features such as power-ups and difficulty level. In retrospect, the code was definitely littered with redundancy and other unappealing characteristics. However, I thought that the game turned out pretty good for a novice.

    Space Invaders

    My second attempt at recreating a game went much more smoothly. This time, I was tasked to code Space Invaders using Java. Graphics was much more challenging this time as I was no longer able to drag and drop shapes onto a form like I could with Visual Basic. However, the whole process was much quicker because I learned what nested loops were, so creating and updating the invaders was much easier. Unfortunately, I did not know about classes or anything related to object-oriented programming, so the game was still riddled with redundancy. I also never figured out a way to make my double buffer work, so the end product had a blinking grid of invaders. Regardless, the game was quite functional and playable.

    What I learned

    On top of natural languages, I finally picked up two programming languages (I no longer remember Visual Basic, though). I felt as if I were an alchmist learning about chemistry since programming completely changed my view of how computer applications worked. Recreating the two games definitely taught me a lot about programming and gave me a great foundation as I continued studying computer science.

    Tools

    Visual Basic, Java, Ctrl+C

    Mind the Gap

    An Android application for London's transit system

    Mind the Gap is an Android application that provides the user with London's transit schedules in real time by querying the London's RESTful TransportforLondon API.

    Phase One

    The first part of the project was mainly to implement the JSON and XML parsers to transform the API responses into something useable. While the coding portion was simple, making sense of the raw API responses was quite the challenge. I was also introduced to JUnit and IntelliJ's powerful debugger, powerful tools to verify code behaviour and to pinpoint bugs.

    Phase Two

    The second part of the project was to create the actual Android application. Using the parsed data from the API responses, the application marks out specific stations and tube lines through a geographic display. The user is further able to query the transit schedules at stations of their choice.

    What I learned

    Having already programmed in Java back in high school, the coding aspect of the project was quite simple. However, I discovered an amazing set of tools that I have never used before such as JUnit and the debugger. I had never created a mobile application before, so seeing the application run natively on my Galaxy Note was definitely a cool experience.

    Tools

    Java, Android, IntelliJ, JUnit

    InsightUBC

    A UBC course and building explorer

    InsightUBC provides a way to effectively query the metadata of various courses and buildings at UBC. It is able to provide information such as course averages and room availability and is also able to create timetables based on the user's input.

    Backend Server

    The initial part of the project was to implement ways for the server to add, remove, and cache UBC metadata given valid zip files. The next part involved finding and transforming the locally cached data on the server based on specific queries provided by the user. For the last part, REST endpoints were created to finalize the API. All of the code were debugged using the Mocha test framework.

    Frontend Webpage

    The server serves a static webpage on which the user is able to browse, search, and refine information regarding courses and buildings. The main goal of the webpage is to transform the user's inputs into valid queries suitable for the backend and to display the response returned by the server.

    What I learned

    This was my first major exposure to asynchronous programming. While Promises and AJAX proved extremely confusing at first, I became extremely comfortable with the asynchronous side of Javascript by the end of the project. I was also able to experiment a bit with some frameworks such as Angular and React, though I settled for vanilla JavaScript and JQuery as the former was completely overkill for my needs.

    Tools

    Typescript, HTML, SCSS, WebStorm, Mocha, Node, JQuery

    GSMS

    A grocery management system

    GSMS is a management system that, among many other things, helps store managers monitor transactions and allows customers to query for various items available at the store. It was a group project between my friends Jonathan, Dalvir, Sonny, and myself.

    Backend Server

    With the growing popularity of Node, it was our primary candidate for the server. We chose Oracle as our database due to the massive support and resources available. The primary role of the server is to transform user data into SQL or PL/SQL statements that are then executed on Oracle.

    Frontend Webpage

    The webpage serves as the control panel that is mainly suitable for the manager. In essence, it serves as a GUI for a typical database, allowing the manager to add, delete, or alter data within the store system. The page can be easily changed to better serve customers by restricting interactions with only views and by limiting queries to only those that do not alter the database.

    What I learned

    The main goal of project was to better familiarize ourselves with databases. We practiced writing and generating dynamic SQL statements that are suitable for Oracle but should be applicable to other database systems. We also experimented with the more powerful PL/SQL. Along the way, we also discovered many other interesting things, including data sanitization to prevent SQL injections.

    Tools

    Typescript, HTML, SCSS, WebStorm, Node, JQuery, Oracle, OracleDB

    Post Me

    Mobile app to post anything locally

    Post Me is a simpler and much more user-friendly take on Craigslist. It allows users to post anything ranging from a sales advertisement for a pair of shoes to a friendly group meet up at the neighbouring park. These posts are then seen by users within the local region and interested participants can conveniently message each other right in the app.

    Backend Server

    The backend is powered by Node using Express as the middleware for routing the HTTP requests sent from Android and Apple mobile devices. It sanitizes the user data such as messages and posts and stores them into the PostgreSQL database server. User verification is done using Google's Firebase by validating user tokens before processing user requests ensure security.

    Mobile application

    The application is minimalistic and intuitive with the intention of being clear and clutter-free. The application features a sign in page for new users and provides a variety of registration options, such as e-mail registration and Google login. Shortly after sign in, the user is presented with a home page with posts from within the local area and has the option to update the user profile, change the settings, manage conversations with other users, and to make a quick post.

    What We learned

    The goal of the project was to expose my partner and I to some of the newer, emerging technologies in the industry. With mobile devices being ubiquitous, we thought that it was a great opportunity to try developing for smartphones and decided on using React Native as it allows us to develop for both Android and iOS with one code base. React Native is extremely similar to React for the browser and gave us quick results when prototyping. We found out that integrating various services such as Google and Facebook login was not simply a plug and play process, as a myriad of problems arose due to incompatibility between the different modules. However, it presented a great chance to practice debugging and problem-solving. In addition, we learned about the importance of security when integrating the login system through reading forums and Firebase's documentations. The project exposed us to some practical insight into what it is like as an actual developer.

    Tools

    React Native, PostgreSQL, Java

    Once Upon a Time

    A farming simulation RPG

    Once upon a time, there lived a person who... what exactly did the person do? That's completely up to you, and you're the narrator of this fairy tale! Utilize the tools you are given and transform the vast, unproductive land into a prospering farmland. Not your type of happily ever after? No problem! Meet and befriend adventurers and villagers in neighbouring towns. Form a powerful party and defeat the bosses in the dungeons nearby. Open up your shop and become a famous merchant among the region by selling weapons, clothes, and desserts you have made.

    What is it?

    Once Upon a Time is a multiplayer-capable farming simulation role-playing game for Windows (and potentially Mac and Linux). The game takes inspiration from three main categories of games I enjoyed as a kid: farming simulation (Harvest Moon), dungeon crawler/exploration (Zelda), and management games (Recettear). As such, iconic game features including transforming the abandoned farm into a prosperous field, befriending and courting villagers, venturing into the dungeon in search of treasure and loot, and managing a successful shop will be focused early in the development and pave the path for future additions.
